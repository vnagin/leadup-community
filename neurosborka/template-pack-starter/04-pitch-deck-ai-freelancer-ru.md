# Питч-дек AI-фрилансера — 10 слайдов

> Используйте на первой встрече с клиентом. Цель — не продать прямо сейчас, а **доказать, что вы понимаете его задачу и можете её решить**. После встречи — выслать индивидуальное КП.

## Слайд 1 — Заголовок

```
[Ваше имя]
AI-автоматизация для [сегмент клиента: e-commerce / клиник / агентств / B2B-продаж — 1 ниша!]

«Превращаю рутину в работающие сценарии за 2-3 недели»

Дата встречи: ___________
Для: [имя компании / контактное лицо]
```

> **Совет:** не пишите «AI для бизнеса». Это никому ни о чём. Лучше: «AI-автоматизация для медицинских клиник», «AI-сценарии для онлайн-школ» — конкретнее = доверие.

## Слайд 2 — В двух словах о вас (30 сек)

```
Кто я
- [предыдущий бэкграунд: 15 лет в [сфере], последние 1-2 года — AI-автоматизация]
- Освоил: n8n, make.com, OpenAI/Anthropic API, retrieval-системы (PGVector / Qdrant)
- Прошёл практическую программу [LeadUp / Нейромастерская] — выпускник 2025

Сделал недавно (3-5 кейсов)
- [клиент 1] — [что сделал] — [результат]
- [клиент 2] — [что сделал] — [результат]
```

> **Совет:** если кейсов нет — поставьте 1-2 «учебных» с чётким описанием, что делали и что получилось. Лучше 2 честных, чем 10 выдуманных.

## Слайд 3 — Что я слышу о вашем бизнесе (10–15 минут разговора)

```
[ЗАПОЛНЯТЬ ПОСЛЕ ВСТРЕЧИ — ключевая часть]

Что вы делаете
- [сегмент рынка, продукт, основная воронка]

Где сейчас узкое место
- [процесс, который занимает много часов / теряет заявки / не масштабируется]

Что бы вы хотели получить
- [результат в человеко-часах / в новых клиентах / в сокращении ошибок]
```

> **Совет:** этот слайд показывайте **только после** того, как клиент рассказал. Это доказательство, что вы слушали. Не угадывайте.

## Слайд 4 — Что обычно работает в AI-автоматизации (краткий обзор)

```
3 типа задач, которые мы сейчас решаем:

1. Поток данных
   Сбор / очистка / маршрутизация (формы → CRM → команды)
2. Содержательная классификация и обогащение
   LLM как «умный сортировщик» (тикеты → категории → команды)
3. Автоматический ответ
   AI-бот / ассистент с retrieval по вашим документам

Все три — построены на n8n / make + LLM API. Без сложной разработки.
```

## Слайд 5 — Конкретный сценарий для вас

```
[ВАРИАНТ ПОД КЛИЕНТА — 1 главный сценарий]

Что предлагаю первым шагом:

[Название сценария]

Кейс шагов:
1. [триггер: webhook / расписание / событие]
2. [обработка: парсинг + LLM-обогащение]
3. [действие: запись в CRM / уведомление / ответ клиенту]

Ожидаемый результат за 2-3 недели:
- [метрика — например: 100% заявок в CRM за 2 минуты]
- [высвобождение времени — например: 30 мин/день менеджера]
```

## Слайд 6 — Технологический стек

```
Что используем
- n8n (или make.com) — оркестратор сценариев
- OpenAI GPT-4o или Anthropic Claude — для классификации и понимания текста
- ваш CRM / БД — встраиваем напрямую через API
- мониторинг — Telegram / Slack alert на ошибки

Где данные
- основная обработка — на вашей инфраструктуре (self-hosted n8n)
- AI-вызовы — через корпоративный API (данные не используются для обучения)
- логи — у вас, под вашим контролем
```

> **Совет:** если клиент чувствительный к данным (медицина, банк) — этот слайд критичен. Не вертитесь, говорите прямо: что где хранится.

## Слайд 7 — План работы

```
Этап 1: Дизайн (1 неделя)
- Карта потоков
- Технические зависимости
- Согласование с вами

Этап 2: Прототип (1-2 недели)
- Сценарий на staging
- Тестирование на ваших данных
- Доработка по обратной связи

Этап 3: Запуск (1 неделя)
- Перенос на прод
- Мониторинг 48 часов
- Обучение команды

Итого: 3-4 недели до результата.
```

## Слайд 9 — Риски и как мы с ними работаем

```
3 риска, которые я открыто называю:

1. AI-классификация может ошибаться (~5% случаев)
   → Митигация: человек-в-цикле для критичных решений

2. Лимиты внешних API (CRM, мессенджер)
   → Митигация: batching + retry, фоновый мониторинг

3. Изменение формата данных от внешних систем (Tilda, фб-формы)
   → Митигация: alert + быстрый фикс на стороне n8n

Если что-то ломается — alert приходит мне в Telegram. Реагирую в течение рабочего дня.
```

> **Совет:** этот слайд парадоксально продаёт сильнее, чем «у нас всё будет идеально». Он строит доверие.

## Шпаргалка по подаче

- **Темп:** 1-2 минуты на слайд. Слайды 3 и 5 — больше времени (главные).
- **Не читайте слайды.** Они — фон. Говорите своими словами.
- **На вопрос «А почему вы?»** — короткая история своего перехода в AI + 1 кейс. Не длинный рассказ.
- **На вопрос про цену** — слайд 8 + готовность раскладку обосновать (см. калькулятор 05).
- **После встречи** — в течение 24 часов: благодарность, краткое summary встречи, конкретное КП с ценой и сроком.
