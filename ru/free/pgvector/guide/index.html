Нейросборка · бесплатный гайд

PGVector + локальный AI: гайд для AI-фрилансера

Короткий гайд для тех, кто хочет сделать первый рабочий RAG-проект на бесплатной базе — без Pinecone, без сложной обвязки и без того, чтобы быть программистом. К гайду идёт пак из 4 готовых n8n-workflow: импортировал, подключил ключи — работает.

Зачем это вам за 5 минут

Если вы берёте заказы на AI-автоматизацию, рано или поздно к вам приходят с одной и той же задачей: «у нас есть гора документов / переписок / регламентов — сделайте, чтобы бот отвечал по ним». Это и есть RAG — поиск по своим данным с ответом от языковой модели.

Большинство гайдов в интернете сразу тащат вас в платный Pinecone и облако, где данные клиента улетают непонятно куда. Для российского заказчика это часто стоп-фактор: и по деньгам, и по 152-ФЗ.

PGVector снимает оба возражения. Это бесплатное расширение для обычного PostgreSQL, которое превращает вашу базу в хранилище для векторного поиска. Postgres можно поднять локально, на своём сервере или в РФ-облаке — данные никуда не уезжают.

Этот гайд объясняет идею простым языком и показывает, с чего начать за один вечер.

RAG простыми словами

Представьте, что языковая модель (GPT, Claude) — это очень начитанный, но забывчивый эксперт. Он знает всё «вообще», но ничего не знает про ваши документы.

RAG — это способ подсунуть эксперту нужную страницу за секунду до ответа:

  1. Складываем документы в базу, разбив на небольшие кусочки («чанки») и переведя каждый в вектор — числовое представление смысла.
  2. Ищем по новому вопросу те кусочки, что ближе всего по смыслу (а не по точному совпадению слов).
  3. Отдаём их модели вместе с вопросом: «ответь, опираясь вот на это».

Вектор — это просто способ для компьютера измерять «похожесть по смыслу». «Договор аренды» и «арендный контракт» окажутся рядом, хотя слова разные. Хранилище таких векторов называется векторной базой. PGVector делает векторной базой ваш обычный Postgres.

Почему PGVector, а не Pinecone

КритерийPGVectorPinecone и облачные аналоги
ЦенаБесплатно (расширение Postgres)От $70/мес за рабочий объём

Порог входаЗнаете SQL на уровне «SELECT» — уже хватитСвой API и термины

PGVector не «лучше во всём» — на десятках миллионов векторов специализированные базы быстрее. Но для 95% задач фрилансера (база знаний компании, поддержка по документам, поиск по архиву) его хватает с запасом. А «бесплатно и данные у клиента» — это часто решающий аргумент на встрече.

4 сценария, которые закрывают почти все заказы

Под каждый — готовый workflow в паке-компаньоне (импортируете в n8n и подключаете ключи):

  1. Загрузка документов в базу (pgvector-ingest-from-folder). Берёте папку с PDF / TXT / MD, прогоняете — всё лежит в PGVector с эмбеддингами. Это фундамент любого проекта.
  2. Поиск по смыслу через API (pgvector-retrieval-basic). Webhook принимает вопрос, возвращает топ-N релевантных кусочков в JSON. Подключаете к любому фронту, боту, таблице.
  3. Telegram-бот «спроси у документов» (pgvector-rag-chat). Вопрос в чат → поиск → ответ модели по найденным фрагментам. Самый частый заказ «под ключ» — и самая зрелищная демка для клиента.
  4. Автотегирование документов (pgvector-metadata-tagging). Модель сама расставляет категории и теги в метаданные. Дальше можно фильтровать поиск «только из договоров», «только за 2025 год» и т.д.

Эти четыре кубика комбинируются почти в любой реальный проект.

С чего начать сегодня вечером

Без программирования, по шагам:

  1. Поднимите Postgres с pgvector. Самый быстрый путь — облачный Postgres, где pgvector уже встроен (Supabase, Neon, Aiven): включается одной галочкой в разделе Extensions. Свой сервер — тоже можно, инструкция в README пака.
  2. Проверьте расширение одной командой: 
    CREATE EXTENSION IF NOT EXISTS vector;
SELECT extname, extversion FROM pg_extension WHERE extname = 'vector';
    Вернулась строчка vector | 0.7.0 (или похожая) — всё готово.
  3. Поставьте n8n (Cloud или Docker) — это конструктор, где workflow собираются из узлов мышкой.
  4. Импортируйте workflow #1 из пака, подключите ключ OpenAI и доступ к Postgres, загрузите 2–3 своих документа.
  5. Импортируйте workflow #3 и задайте боту вопрос по этим документам. Это ваш первый работающий RAG — за один вечер.
Не хотите OpenAI? В стикерах внутри workflow показаны замены на Anthropic или локальную Ollama.

Чек-лист «я готов брать заказ на RAG»

  • Подняли Postgres с pgvector и проверили расширение
  • Загрузили свои документы в базу (workflow #1)
  • Получили осмысленный ответ от бота по своим данным (workflow #3)
  • Понимаете, где живут данные клиента и как объяснить это с точки зрения 152-ФЗ
  • Можете показать живую демку за 10 минут на встрече

Отметили всё — вы уже сильнее большинства, кто «просто умеет звать ChatGPT».
