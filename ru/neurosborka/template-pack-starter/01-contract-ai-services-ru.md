# Договор оказания AI-услуг

**ВАЖНО:** Это упрощённый шаблон. Не используйте его без адаптации в проектах с бюджетом 500k+ ₽, регулярными платежами, использованием персональных данных клиентов клиента, или экспортом данных за пределы РФ. В таких случаях — проконсультируйтесь с юристом.

## Шаблон

```
ДОГОВОР № __________
оказания услуг по разработке AI-решений

г. ____________                                         «___» ___________ 20___ г.

Исполнитель: __________________________________________________________________
(ФИО / самозанятый / ИП / ЮЛ; ИНН _______________; адрес: ____________________)

Заказчик: _____________________________________________________________________
(ФИО / ИП / ЮЛ; ИНН _______________; адрес: ____________________)

именуемые в дальнейшем «Стороны», заключили настоящий Договор о нижеследующем:

1. ПРЕДМЕТ
1.1. Исполнитель обязуется оказать Заказчику услуги по разработке AI-решения (далее — «Решение») в соответствии с Техническим заданием (Приложение №1, далее — «ТЗ»), а Заказчик — принять и оплатить эти услуги.
1.2. Состав Решения, технологии, ожидаемые результаты — указаны в ТЗ.

2. СРОКИ И ЭТАПЫ
2.1. Срок оказания услуг: с «___» __________ по «___» __________ 20___ г.
2.2. Этапы и сроки промежуточных результатов — в Приложении №2 (Календарный план).

3. ЦЕНА И ПОРЯДОК ОПЛАТЫ
3.1. Стоимость услуг по настоящему Договору составляет _________________ рублей (без НДС / с учётом НДС — нужное оставить).
3.2. Оплата производится в порядке: ____% — аванс при подписании, ____% — по итогу приёмки.
3.3. Способ оплаты: банковский перевод на реквизиты Исполнителя (Приложение №3).
3.4. Датой оплаты считается дата зачисления средств на расчётный счёт Исполнителя.

4. ПЕРЕДАЧА И ПРИЁМКА
4.1. По завершении этапа Исполнитель направляет Заказчику Решение и Акт оказанных услуг.
4.2. Заказчик в течение ____ рабочих дней подписывает Акт или направляет мотивированный отказ с перечнем замечаний.
4.3. При отсутствии подписанного Акта и отсутствии мотивированного отказа в указанный срок — услуги считаются принятыми.

5. ИНТЕЛЛЕКТУАЛЬНАЯ СОБСТВЕННОСТЬ
5.1. Исключительные права на разработанное Решение в полном объёме передаются Заказчику с момента полной оплаты.
5.2. Исполнитель вправе использовать **обезличенные** части Решения (общие куски кода, шаблоны промптов, методологию) в собственных работах и в портфолио, если иное не оговорено в ТЗ.
5.3. Право Исполнителя упоминать факт сотрудничества в портфолио — указано в п. 5.4.
5.4. Заказчик: ☐ согласен / ☐ не согласен на упоминание факта сотрудничества Исполнителем в публичных кейсах. (Имя/логотип Заказчика — только при отдельном письменном согласии.)

6. КОНФИДЕНЦИАЛЬНОСТЬ
6.1. Стороны соглашаются не разглашать конфиденциальную информацию, полученную в ходе исполнения Договора. Подробности — в отдельном NDA (Приложение №4), если стороны его подписывают.
6.2. Под конфиденциальной информацией понимается: данные клиентов Заказчика, бизнес-процессы, не публичные документы, ключи API сторонних сервисов.

7. ПЕРСОНАЛЬНЫЕ ДАННЫЕ (152-ФЗ, если применимо)
7.1. Если в ходе оказания услуг Исполнитель обрабатывает персональные данные физических лиц от имени Заказчика — Стороны подписывают отдельное Поручение на обработку ПДн (Приложение №5).
7.2. Без такого Поручения — Исполнитель не получает доступ к ПДн.

8. ОТВЕТСТВЕННОСТЬ
8.1. За нарушение сроков оплаты — пеня 0,1% от просроченной суммы за каждый день, но не более 10% от общей стоимости.
8.2. За нарушение сроков оказания услуг — Исполнитель устраняет просрочку в разумный срок без штрафа, при условии своевременного информирования Заказчика.
8.3. **Ограничение ответственности:** ответственность Исполнителя ограничена суммой полученной оплаты. Исполнитель не несёт ответственности за упущенную выгоду Заказчика.

9. ПРАВО НА AI-СГЕНЕРИРОВАННЫЙ КОНТЕНТ
9.1. Если в составе Решения используется AI-сгенерированный контент (тексты, изображения, код) — Исполнитель гарантирует, что использовал инструменты с лицензией, разрешающей коммерческое использование выходов (OpenAI, Anthropic, Mistral и т.п. — на условиях их API).
9.2. Заказчик понимает, что AI-выходы могут содержать неточности и обязуется самостоятельно проверять корректность фактов, юридических формулировок, цифр.

10. ФОРС-МАЖОР, РАСТОРЖЕНИЕ, ПРОЧЕЕ
10.1. Стандартные условия (форс-мажор, разрешение споров — переговоры → суд по месту Истца, изменения — только в письменной форме).
10.2. Договор может быть расторгнут по соглашению Сторон или в одностороннем порядке с уведомлением за ___ дней.

11. РЕКВИЗИТЫ И ПОДПИСИ

Исполнитель                                  Заказчик
________________ / ___________ /             ________________ / ___________ /
МП (если есть)                                МП (если есть)
```

## Чек-лист использования

- [ ] Заполнить ИНН, адреса, реквизиты Сторон.
- [ ] Подготовить Приложение №1 (ТЗ — см. шаблон 03).
- [ ] Подготовить Приложение №2 (Календарный план).
- [ ] Если есть NDA — приложить шаблон 02.
- [ ] Если обрабатываются персональные данные — добавить Поручение по 152-ФЗ.
- [ ] Указать процент аванса/постоплаты (по умолчанию: 50/50; для нового клиента — 70/30 с предоплатой).
- [ ] Зафиксировать пункт 5.4 (упоминание в портфолио — да/нет).

## Что не покрывает этот шаблон

- Регулярные платежи / подписочная модель — нужно отдельное приложение.
- Передача данных за пределы РФ — отдельный пункт + 152-ФЗ.
- Гарантии/SLA на работу AI-системы — нужно проектировать индивидуально.
- Споры о праве на AI-сгенерированный код в EU/US-юрисдикциях — нужен юрист.
